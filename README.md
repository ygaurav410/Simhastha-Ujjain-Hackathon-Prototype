# Project Title - NaviMitra - Smart Event Navigation Platform
# Team ID - TH1364

## 1. Overview

NaviMitra is a comprehensive navigation and safety platform designed for managing large-scale events, such as the Kumbh Mela. It provides real-time crowd monitoring, efficient emergency response systems, and AI-powered routing to enhance the safety and experience of millions of attendees. The platform supports multiple user roles, including civilians, administrators, and authorities, ensuring synchronized data and coordinated efforts across all stakeholders.

## 2. Problem & Solution

*   **Problem Statement:** Large-scale events present significant challenges in managing vast crowds, ensuring efficient navigation, responding to emergencies promptly, and maintaining overall safety. Traditional methods often fall short in providing real-time insights and coordinated responses, leading to potential hazards and logistical complexities.

*   **Solution:** NaviMitra addresses these challenges by offering a digital, integrated platform. It leverages real-time data for crowd monitoring, provides AI-powered routing to optimize movement, and streamlines emergency reporting and response. This centralized system enhances situational awareness for authorities and empowers attendees with critical information and navigation tools, ultimately improving safety and operational efficiency.

## 3. Logic & Workflow

*   **Data Collection:**
    *   **User-Generated:** Civilian users can submit emergency reports and posts (e.g., lost persons, medical emergencies, security issues) through intuitive forms, including location details and contact information.
    *   **System-Generated:** The platform conceptually integrates with IoT sensors and AI-enabled cameras for real-time crowd density data. Alerts are generated by admin/authority users to disseminate critical information.
    *   **Location Data:** User locations, amenity points, and incident coordinates are collected to provide context-aware services.

*   **Processing:**
    *   **Real-time Synchronization:** Data is synchronized across all user panels in real-time, ensuring that all stakeholders have access to the most current information.
    *   **State Management:** A global state management system (`globalState`) facilitates seamless data flow and updates across different components and user interfaces.
    *   **Role-Based Access:** Data is filtered and presented based on the user's assigned role (civilian, admin, authority), ensuring appropriate access and functionality.
    *   **AI Integration (Conceptual):** The system is designed to incorporate AI algorithms for analyzing crowd movement patterns and optimizing navigation routes.

*   **Output:**
    *   **Interactive Maps:** Visual representation of user locations, alerts, amenities, and crowd density heatmaps.
    *   **Dashboards:** Centralized dashboards for admin and authority users displaying key metrics, active alerts, and pending emergency reports.
    *   **Alerts & Notifications:** Real-time alerts and safety advisories delivered to relevant users.
    *   **Emergency Feeds:** Categorized lists of emergency posts and reports for community awareness and official response.
    *   **Navigation Guidance:** Optimized routes and directions to points of interest or safety zones.

*   **User Side (Civilian):**
    *   **Authentication:** Secure login and signup, including social login options.
    *   **Navigation:** Access to interactive maps, smart navigation, and route suggestions.
    *   **Emergency Reporting:** Ability to report various types of emergencies with detailed information and location.
    *   **Information Access:** View real-time alerts, find nearby amenities (hospitals, police stations, food courts), and browse community emergency posts.
    *   **Offline Mode:** Basic GPS navigation and access to cached data even without internet connectivity.

*   **Admin Side:**
    *   **Command Center:** Overview dashboard with critical statistics and system health.
    *   **Emergency Management:** Monitor, assign, and update the status of emergency reports.
    *   **Alert Management:** Create and broadcast system-wide alerts based on severity and type.
    *   **Crowd Monitoring:** Visualize crowd density on heatmaps to identify potential congestion areas.
    *   **Database Viewer:** Direct access to view and manage data within the Firebase collections for debugging and oversight.
    *   **Resource Deployment:** Conceptual tools for deploying response teams to incidents.

## 4. Tech Stack

*   **Frontend:**
    *   **React.js:** A JavaScript library for building user interfaces.
    *   **TypeScript:** A typed superset of JavaScript that compiles to plain JavaScript.
    *   **Vite:** A fast build tool that provides a lightning-fast development experience.
    *   **Tailwind CSS:** A utility-first CSS framework for rapidly building custom designs.
    *   **Lucide React:** A collection of open-source icons.

*   **Backend/Database:**
    *   **Firebase:** Used for authentication, Firestore (NoSQL database for real-time data synchronization), and potentially other Firebase services like Storage and Analytics.
    *   **Supabase:** Utilizes PostgreSQL for relational data management, offering real-time capabilities, authentication, and serverless Edge Functions.

*   **Mapping:**
    *   **Conceptual Mapping API:** Designed to integrate with a mapping service (e.g., Google Maps API) for interactive map functionalities.

## 5. Future Scope

The NaviMitra platform has significant potential for expansion and enhancement:

*   **Advanced AI Integration:** Implement predictive analytics for crowd movement, real-time risk assessment, and optimized resource allocation to prevent incidents before they occur.
*   **Real-world IoT & Sensor Integration:** Connect with actual IoT sensors, CCTV cameras, and drone feeds for highly accurate and granular crowd data, environmental monitoring, and incident verification.
*   **Native Mobile Applications:** Develop dedicated iOS and Android applications to leverage device-specific features like enhanced GPS, push notifications, and offline capabilities more robustly.
*   **Enhanced Communication:** Introduce real-time chat functionalities between civilian reporters, authorities, and response teams, and potentially live video streaming from incident sites.
*   **Integration with Official Emergency Services:** Establish direct API integrations with government emergency services (police, medical, fire) for seamless dispatch and coordination.
*   **Multilingual Support:** Expand language options to cater to a diverse audience, ensuring accessibility for all attendees.
*   **Gamification & Incentives:** Implement features to encourage civilian participation in reporting and community safety, such as points or badges for verified contributions.
*   **Post-Event Analytics:** Develop comprehensive tools for analyzing event data to generate insights for future event planning and disaster preparedness.

## Getting Started

### Prerequisites

- Node.js (v18 or higher)
- npm or yarn package manager

### Installation

1. Clone the repository:
   ```bash
   git clone <repository-url>
   cd navimitra
   ```

2. Install dependencies:
   ```bash
   npm install
   ```

3. Start the development server:
   ```bash
   npm run dev
   ```

4. Open your browser and navigate to `https://stackblitz.com/~/github.com/ygaurav410/Simhastha-Ujjain-Hackathon-Prototype`
   On the right side of the screen, a panel with a running prototype will automatically start, providing easier access to the codebase and an editor. You can navigate through the app from there, or you can open it in a new tab for better viewing via a button in the top right corner.
